CÓMO USAR EXTENDS Y BLOCK EN JADE

Ya aprendimos como Importar en Jade ahora iremos un paso más allá.

Extends

Se basa en la idea de tener un archivo base y este ser reusado en todas nuestras vistas.

Block

Define un bloque de código para que luego pueda ser cambiado.

A continuación un ejemplo.

Este será nuestro archivo base.jade (Normal)


!!! 5
html(lang="es")
	head
		meta(charset="UTF-8")
		title Hola Mundo
	body
		.wrapper
			.header Header
		  article.main
		    p Lorem ipsum dolor sit amet, consectetur adipisicing elit. Nisi, est.
		  aside.aside.aside-1 Aside 1
		  aside.aside.aside-2 Aside 2
		  footer.footer Footer

1) Ahora definiremos el código con Block | base.jade (Modificado)


!!! 5
html(lang="es")
	head
		meta(charset="UTF-8")

		block titulo
			title Hola Mundo

	body
		.wrapper

			block cabecera
				.header Header

		  article.main
		    p Lorem ipsum dolor sit amet, consectetur adipisicing elit. Nisi, est.
		  aside.aside.aside-1 Aside 1
		  aside.aside.aside-2 Aside 2
		  footer.footer Footer

2) Una vez definidos los block procedemos a usar el extends en un archivo nuevo | index.jade


extends base.jade
block titulo
	title Escuela Digital

En el punto "2" el archivo index.jade hacemos el llamado al archivo base.jade con toda su estructura pero solo cambiará el block titulo.

En el caso del block cabecera se imprimirá tal cual por que en el archivo index.jade no se produjo ningún cambio a este.

3) Y así se mostrará el html index.html 

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Escuela Digital</title>
  </head>
  <body>
  	<div class="wrapper">
	  <header class="header">Header</header>
	  <article class="main">
	    <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit. Nisi, est.</p>
	  </article>
	  <aside class="aside aside-1">Aside 1</aside>
	  <aside class="aside aside-2">Aside 2</aside>
	  <footer class="footer">Footer</footer>
	</div>
  </body>
</html>

Recomendación

  • Tener mucho cuidado con la indentación por que Jade es muy sensible en ello.
  • La definición con block es posible hacerla con muchos niveles de jerarquía.
  • Así como vimos en el punto 2. Pueden crear sus propias vistas para los archivos que ustedes quieran como (hola.jade, blog.jade, contacto.jade, etc.)
